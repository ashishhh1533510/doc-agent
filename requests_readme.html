Requests HTTP Library

This library provides a simple and elegant API for making HTTP requests in Python.

Overview

This project is organized into several modules, each responsible for a specific aspect of handling HTTP requests and responses.

  • requests: The main package, providing the high-level API for making requests.
  • requests._internal_utils: Contains internal utility functions used throughout the library.
  • requests._types: Defines type hints and interfaces for various request-related components.
  • requests.adapters: Manages the transport layer for requests, handling connections and responses.
  • requests.api: Exposes the main request functions like get, post, etc.
  • requests.auth: Provides classes for handling different authentication schemes.
  • requests.certs: Provides access to SSL certificates.
  • requests.compat: Contains compatibility code for different Python versions.
  • requests.cookies: Manages HTTP cookies and provides utilities for handling them.
  • requests.exceptions: Defines custom exception classes for various error conditions.
  • requests.help: Includes utility functions for gathering information for bug reports.
  • requests.hooks: Implements the hook system for extending request behavior.
  • requests.models: Defines the core Request and Response objects.
  • requests.sessions: Manages persistent connections and session-level settings.
  • requests.structures: Contains custom data structures like CaseInsensitiveDict.
  • requests.utils: Offers various utility functions for URL manipulation, header parsing, and more.

Usage / API Reference

This section lists key functions and classes available in the requests library.

requests Package

  • check_compatibility(): Checks for compatibility with underlying libraries.

requests.api Module

  • request(method, url, **kwargs): The low-level request method.
  • get(url, params=None, **kwargs): Sends a GET request.
  • options(url, **kwargs): Sends an OPTIONS request.
  • head(url, **kwargs): Sends a HEAD request.
  • post(url, data=None, json=None, **kwargs): Sends a POST request.
  • put(url, data=None, **kwargs): Sends a PUT request.
  • patch(url, data=None, **kwargs): Sends a PATCH request.
  • delete(url, **kwargs): Sends a DELETE request.

requests.models Module

  • Request: A user-created :class:Request <Request> object.
    • prepare(): Prepares the request to be sent.
  • PreparedRequest: The fully mutable :class:PreparedRequest <PreparedRequest> object.
    • prepare(): Prepares the request.
    • copy(): Creates a copy of the prepared request.
    • prepare_method(method): Prepares the request method.
    • prepare_url(url, params): Prepares the request URL.
    • prepare_headers(headers): Prepares the request headers.
    • prepare_body(data, files): Prepares the request body.
    • prepare_content_length(content_length): Prepares the content length.
    • prepare_auth(auth, url, verify): Prepares the authentication.
    • prepare_cookies(cookies): Prepares the request cookies.
    • prepare_hooks(hooks): Prepares the request hooks.
  • Response: The :class:Response <Response> object, containing a response from the server.
    • ok: Boolean indicating if the status code is less than 400.
    • is_redirect: Boolean indicating if the response is a redirect.
    • is_permanent_redirect: Boolean indicating if the response is a permanent redirect.
    • next: Returns the next response in a redirect chain.
    • apparent_encoding: The apparent encoding of the response content.
    • iter_content(chunk_size=1, decode_unicode=False): Iterates over the response content.
    • iter_lines(chunk_size=512, decode_unicode=False): Iterates over the response lines.
    • content: The response body as bytes.
    • text: The response body as a string, decoded using the detected encoding.
    • json(): Decodes the response body as JSON.
    • links: A dictionary of link headers.
    • raise_for_status(): Raises an HTTPError if the HTTP request returned an unsuccessful status code.
    • close(): Closes the response and releases the connection.

requests.exceptions Module

This module defines various exception classes for handling errors:

  • RequestException: Base exception for all exceptions.
  • InvalidJSONError: A JSON error occurred.
  • HTTPError: An HTTP error occurred.
  • ConnectionError: A Connection error occurred.
  • ProxyError: A proxy error occurred.
  • SSLError: An SSL error occurred.
  • Timeout: The request timed out.
  • ConnectTimeout: The request timed out while trying to connect to the remote server.
  • ReadTimeout: The server did not send any data in the allotted amount of time.
  • URLRequired: A valid URL is required to make a request.
  • TooManyRedirects: Too many redirects.
  • MissingSchema: The URL scheme (e.g. http or https) is missing.
  • InvalidSchema: The URL scheme provided is either invalid or unsupported.
  • InvalidURL: The URL provided was somehow invalid.
  • InvalidHeader: The header value provided was somehow invalid.
  • InvalidProxyURL: The proxy URL provided is invalid.
  • ChunkedEncodingError: The server declared chunked encoding but sent an invalid chunk.
  • ContentDecodingError: Failed to decode response content.
  • StreamConsumedError: The content for this response was already consumed.
  • RetryError: Custom retries logic failed
  • UnrewindableBodyError: Requests encountered an error when trying to rewind a body.
